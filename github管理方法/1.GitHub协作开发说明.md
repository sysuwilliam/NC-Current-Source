# GitHub 协作开发说明

## 适用场景
这份说明分成两部分：

1. 仓库拥有者如何把队友加入协作
2. 队友如何在本地使用 Git 参与开发

---

## 一、推荐协作方式

对于小团队比赛项目，最简单稳定的方式是：

1. 作为仓库拥有者，在 GitHub 上把队友加入仓库协作者
2. 每个队友把仓库克隆到自己电脑
3. 每个人先同步最新代码，再修改，再提交，再推送
4. 尽量做到“一个人负责一部分内容”，减少同时修改同一个文件

如果项目规模还不大，这种方式已经够用了，不一定一开始就上复杂的分支管理流程。

---

## 二、仓库拥有者如何邀请队友加入协作

### 1. 进入仓库页面

打开 GitHub 仓库页面：

`https://github.com/sysuwilliam/NC-Current-Source`

### 2. 打开仓库设置

在仓库页面上方找到：

`Settings`

### 3. 找到协作者管理入口

在左侧菜单中找到类似下面的入口：

`Collaborators`

有时它会出现在：

`Settings -> Collaborators and teams`

### 4. 邀请队友

点击：

`Add people`

然后输入队友的 GitHub 用户名，发送邀请。

### 5. 队友接受邀请

队友需要登录 GitHub，在通知或邮箱中接受邀请。接受之后，他们就能向这个仓库直接推送代码。

---

## 三、每个队友开始前要准备什么

每个队友本地电脑需要准备：

1. 一个 GitHub 账号
2. 已安装 Git
3. 能正常登录 GitHub
4. 本地有一个用于存放项目的文件夹

### 检查 Git 是否安装成功

在终端中输入：

```bash
git --version
```

如果能看到版本号，就说明 Git 已安装。

---

## 四、首次把仓库下载到本地

### 方式 1：使用 SSH 克隆

如果已经配置好 SSH key，执行：

```bash
git clone git@github.com:sysuwilliam/NC-Current-Source.git
```

### 方式 2：使用 HTTPS 克隆

如果还没配置 SSH，也可以执行：

```bash
git clone https://github.com/sysuwilliam/NC-Current-Source.git
```

### 进入项目目录

```bash
cd NC-Current-Source
```

---

## 五、第一次配置自己的 Git 身份

每个队友第一次使用 Git 时，建议先配置用户名和邮箱：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的GitHub邮箱"
```

例如：

```bash
git config --global user.name "Liu Xinya"
git config --global user.email "example@qq.com"
```

这样以后每次提交，都会记录是谁提交的。

## 六、日常协作的标准流程

以后每次开发，建议都按这个顺序：

1. 先进入项目目录
2. 先拉取最新代码
3. 再开始修改
4. 修改后查看变更
5. 提交到本地仓库
6. 推送到 GitHub

对应命令如下：

```bash
git pull origin main
```

修改文件后：

```bash
git status
git add .
git commit -m "写清楚这次改了什么"
git push origin main
```

---

## 七、每条命令是什么意思

### 1. `git status`

查看当前有哪些文件被修改、哪些文件还没有加入暂存区。

```bash
git status
```

### 2. `git add .`

把当前目录下修改过的文件加入暂存区，准备提交。

```bash
git add .
```

如果不想一次加入所有文件，也可以单独加入某个文件：

```bash
git add README.md
```

### 3. `git commit -m "..."`

把这次修改正式记录到本地 Git 历史中。

```bash
git commit -m "完成串口调试框架"
```

提交信息要尽量具体，不要只写“修改了一下”。

### 4. `git push origin main`

把本地提交推送到 GitHub 远程仓库。

```bash
git push origin main
```

### 5. `git pull origin main`

把 GitHub 上最新的代码拉到本地，并合并进当前分支。

```bash
git pull origin main
```

## 八、8建议遵守的协作习惯

### 1. 开始写代码前先 `pull`

不要一上来就改。先执行：

```bash
git pull origin main
```

这样可以减少冲突。

### 2. 一次只做一类修改

例如这次只改：

- 串口通信
- ADC 采样
- LCD 显示
- 文档说明

不要把很多无关内容混在一个提交里。

### 3. 提交信息写清楚

推荐写法：

```bash
git commit -m "新增电流设定串口指令解析"
git commit -m "修复 ADC 采样值缩放错误"
git commit -m "补充硬件连接说明文档"
```

### 4. 尽量避免多人同时改同一个文件

比如：

- 一个人负责 `README`
- 一个人负责控制算法
- 一个人负责显示或调试模块

这样最省事。

## 九、如果推送失败怎么办

最常见的原因是：你本地代码不是最新的，别人已经先推送了。

这时通常会提示你先拉取远程更新。

解决步骤：

```bash
git pull origin main
```

如果没有冲突，拉完之后再执行：

```bash
git push origin main
```

---

## 十、如果出现冲突怎么办

冲突的典型场景是：

1. 你修改了某个文件
2. 队友也修改了同一个文件的同一部分
3. 你执行 `git pull` 时，Git 不知道该保留谁的内容

### 冲突时 Git 会在文件里标出内容

文件中可能会出现类似内容：

```text
<<<<<<< HEAD
这是你的修改
=======
这是队友的修改
>>>>>>> 分支信息
```

### 解决方法

1. 打开冲突文件
2. 手动决定保留哪些内容
3. 删除 `<<<<<<<`、`=======`、`>>>>>>>` 这些标记
4. 保存文件
5. 重新提交

命令一般是：

```bash
git add .
git commit -m "解决合并冲突"
git push origin main
```

## 最小命令集合

```bash
git clone 仓库地址
cd 仓库目录
git pull origin main
git status
git add .
git commit -m "本次修改说明"
git push origin main
```

## 如何查看历史记录

### 查看提交记录

```bash
git log --oneline
```

这样可以快速看到谁提交了什么。

### 查看具体改动

```bash
git diff
```

这个命令可以查看当前还没提交的修改。

---

## 如果误操作了怎么办

### 场景 1：文件改乱了，但还没提交

先不要慌，可以先用：

```bash
git status
```

看看哪些文件被改了。

### 场景 2：已经提交了，但还没推送

先找队友或你确认一下，再处理。因为 Git 的撤销方式很多，操作不当可能会把内容弄丢。

### 建议

对于新手，遇到不确定的情况时，不要随便执行危险命令，例如：

```bash
git reset --hard
git clean -fd
```

这类命令可能直接删除本地修改。

## 推荐的实际上手流程

你可以把下面这段直接发给队友：

### 队友加入项目步骤

1. 注册并登录 GitHub
2. 把 GitHub 用户名发给队长
3. 接受仓库协作邀请
4. 安装 Git
5. 执行 `git clone` 把仓库拉到本地
6. 进入项目目录后先执行 `git pull origin main`
7. 修改自己负责的内容
8. 执行 `git add .`
9. 执行 `git commit -m "说明本次修改"`
10. 执行 `git push origin main`

## 总结

对你们现在这个仓库，最实用的协作方案就是：

1. 先把队友加成 Collaborator
2. 队友克隆仓库到本地
3. 每次开发前先 `pull`
4. 每次开发后 `commit` 和 `push`
5. 尽量按模块分工，减少同时修改同一个文件
6. 后期代码复杂后，再逐步切换到“功能分支 + Pull Request”模式
