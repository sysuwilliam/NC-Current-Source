# 总体说明
- 电源树
  ```
   35V 输入
    ├── 可调 Buck：V+，给负载 + MOS 恒流功率级，5~30V
    └── 固定 Buck：5V
        ├── 串口屏
        ├── OPA197 运放
        └── LDO 3.3V
              ├── STM32
              ├── DAC
              ├── TS5A3159A
              └── 比较器/ADC参考相关小信号电路
  ```
- 恒流控制逻辑
  ```
  硬件恒流环：运放 + MOS：毫秒甚至更快，负责稳电流
  buck余量控制环：MCU + Buck：几十毫秒到几百毫秒更新一次，负责控发热
  【后期优化再加】电流校准环：MCU 读取 Iactual，发现 Iactual 和 Iset 有小偏差 -> 很慢很慢地微调 DAC_CH2
  ```
  流程：
  ```
  用户设定 Iset
      ↓
  MCU 输出 DAC_CH2 = Iset × Rs
      ↓
  运放 + MOS + Rs 硬件恒流
      ↓
  MCU 读取 Vsense，显示 Iactual，做保护判断
      ↓
  MCU 读取 VOUT+、VOUT-、Vsense
      ↓
  计算 VMOS = VOUT- - Vsense
      ↓
  慢速调整 DAC_CH1，改变 Buck 输出 V+
  ```

# 计算与采样
- 采样点
  - PA0 = Vsense，采样电阻电压，算电流
  - PA1 = OUT+ / V+，Buck 输出，负载上端
  - PB0 = OUT- / MOS Drain，负载下端
  

- 输出电流计算
  ```
  DAC_ISET：0~1.0V
  Iout = DAC_ISET / 2Ω
  ```
- DAC->可调buck计算
  ```
  V_OUT ≈ 30.98 - 10 * V_DAC
  DAC_CH1 工作范围大约：0.1V ~ 2.6V (对应 30V ~ 5V)
  ```
- 负载电阻计算
  ```
  VOUT+ = ADC_VOUT+ × 11 × 校准系数
  VOUT- = ADC_VOUT- x 11 × 校准系数
  Vload = VOUT+ - VOUT-
  Rload = Vload / Iout
  ```
- mos压降计算
  ```
  VMOS​=VOUT-​ - Vsense​
  ```

# 功能实现
- 过流保护实现
  - 流程：
    ```
    Vsense → MCU ADC

    MCU_GPIO → 模拟开关IN & 可调 Buck EN

    MCU → 串口屏
    ```
  - 示例代码：
    ```
    if (Vsense > 1.10V) {
      SW_IN = 0;          // 模拟开关切到 GND，Iset=0
      DAC_ISET = 0;       // DAC 输出也归零
      BUCK_EN = 0;        // 关闭可调 Buck
      fault_oc = 1;       // 锁存过流故障
      show_alarm();       // LCD 显示过流
    }
    // 注意这里的 alarm_flag 要锁存，不要电流一降下来就自动清除；可以直到用户按复位按键（不是单片机上的）再清...
    ```
- 短路保护实现
  - 判定
    ```
    set > 20mA
    Iactual ≈ Iset
    Vload < 0.2V ~ 0.5V
    持续 50ms~200ms
    (电流能达到设定值，但负载电压接近 0)
    ```
  - 执行
    ```
    把 Buck 输出 V+ 降到较低值  (V+​ ≈ Iset * ​Rs ​+ VMOS余量​)
    LCD 显示“短路限流”
    ```
  - 解除
    ```
    短路解除后，负载电压不再接近 0，程序重新进入正常 Buck 调节逻辑，慢慢把 V+ 升到需要的值
    这个保护不需要锁存，可以自动恢复
    ```
- 开路保护
  - 判定
    ```
    Iset > 20mA
    Iactual < 5mA
    Buck 已经升到较高值 / 或 V+ 接近当前目标值
    持续 200ms
    => 开路
    ```
  - 执行
    ```
    1. 模拟开关 IN 拉低 → Iset 接 GND
    2. DAC_ISET = 0
    3. 可调 Buck EN 关闭，或者 V+ 降到最低
    4. LCD 显示“负载开路”
    5. 等待用户接好负载后手动清除报警
    ```
  - 解除
    ```
    按ALARM CLEAR按键 （能混用吗？）
    ```

# 交互内容
- 外设
  ```
  1 个 带按压旋转编码器
  1 个 OUTPUT / STOP 按键
  ```

  - 开启输出：按OUTPUT
  - 触发过流保护/开路保护后的恢复：按ALARM CLEAR
  - ALARM CLEAR 状态机
    ```
    正常输出状态
        ↓
    检测到过流 / 开路
        ↓
    FAULT 状态：
        SW_IN = 0
        DAC_ISET = 0
        BUCK_EN = 0
        显示报警
        ↓
    用户按 ALARM_CLEAR
        ↓
    IDLE 待机状态：
        报警清除
        输出仍关闭
        ↓
    用户再按 OUTPUT/STOP
        ↓
    重新启动输出
    ```

# 原理图元件说明
### R5（运放IN-与Rs上端之间）（对应开源工程中R17）
- 取47Ω（或0Ω/100Ω）不影响直流精度，因为运放输入电流非常小，R5上几乎没有压降
- 用处：
  - 调试时方便断开：万一反馈接错、噪声很大，拆掉 R17 就能把运放反相端和采样点隔离开，方便定位问题
  - 隔离采样电阻大电流节点上的尖峰噪声

### R4（R_GATE）（对应开源工程中R16）
- 默认 47Ω ，如果响应太慢可以改 22Ω；如果振荡可以改 100Ω

### R1(COM到IN+)（对应开源工程中R15）
- DAC 到同相端最好还是留一个串联电阻，用来隔离 DAC、模拟开关和输入滤波

### R2（采样电阻）
- 2Ω 0.1% 
- ≤25ppm/℃
- ≥2W，最好 3W
- 2512 或四端 Kelvin 采样电阻

### C2（运放IN-到OUT）（对应开源工程中C23）
- 取值：不焊 / 330pF / 1nF 
- 开源工程原来用 C23=10nF，是配合 LMH6642、R15=2k、R17=1k、R19=2.49Ω 那套反相求和结构的，现在结构已经变了，不能照抄。调试时先不焊 C23；如果示波器看到 Vsense 有持续高频振荡，再往空焊盘上加 330pF。没有振荡就一直不焊。若加了还振，换 1nF；如果响应太慢，再拆掉。

### C27/C28（VOUT+-采样）
- C越小响应越快，越大滤波效果越好
- 先取22nF，如果buck开关噪声明显、显示抖，再改大到 100nF
- PCB布局注意：
  ```
  R25 和 C27 尽量靠近 MCU ADC 引脚
  R19/R24 靠近被采样的 VOUT+ 节点
  （VOUT-采样电路同理）
  ```

### C35（V_SENSE滤波）
- 暂用 10nF，如果 ADC 抖得厉害，可以把 C35 改到 22nF 或 100nF

### mos管
- 封装：TO220是最好散热的，但是是直插的体积较大，而且考虑到我们后续板子装在外壳里，所以选了散热和美观平衡一些的TO252（型号是SUD50N06-09L-TP），没用开源工程的PQFN封装。但现在散热还是很依赖pcb设计，而且要确保一级buck使mos压降一直控制在2~4V
- PCB
  ```
  MOS Drain 大焊盘向外铺大铜皮
  上下层都铺铜
  用很多过孔把热导到底层
  底层不要被 LCD 完全贴住
  外壳对应位置开散热孔
  ```

### DAC芯片
- 初始化程序
  ```
  DAC8562_Write24(0x380001);   // 开启内部 2.5V reference，gain 默认变成 2 （2.5V*2=5.0V）
  DAC8562_WriteB(0);           // DAC_CH2 = 0V，电流目标清零
  DAC8562_WriteA(initial_code); // DAC_CH1 给 Buck 初始控制值
  ```

# PCB布局与数模地连接（AI提供，仅供参考）
- 采用二层 PCB，不照搬开源工程的四层板数模地划分方式。为了降低布线复杂度，同时保证电流采样精度，原理图中只保留两类地：
`GND：功率地 + 数字地 + 系统公共地`
`AGND：模拟采样参考地`


### 1. GND 的范围
以下部分接 `GND`：

```text
35V 输入负端
输入保护电路
输入 TVS
输入滤波电容
固定 5V Buck
可调 Buck
MOS + Rs 主功率回流
STM32 核心板
串口屏
调试串口
按键
编码器
LED
其他数字外设
```

主功率电流路径为：

```text
VOUT+ → 负载 → MOS → Rs → GND
```

这条路径要：

```text
走粗线 / 大铜皮
路径尽量短
不要绕到 DAC、OPA、ADC 采样区域下面
```

### 2. AGND 的范围

以下部分接 `AGND`：

```text
OPA197 运放地
DAC8562 地
TS5A3159A 模拟开关地
VOUT+ / VOUT- 采样分压下端
Vsense ADC 滤波电容下端
DAC 输出滤波电容下端
OPA197 相关去耦电容地
模拟输入滤波电容地
```

AGND 区域应该尽量小，集中放在：

```text
OPA197
DAC8562
TS5A3159A
采样分压电阻
采样电阻 Rs 附近
```

不要让下面这些回流经过 AGND：

```text
Buck 开关电流
输入电容充放电电流
MOS 主功率电流
串口屏电流
按键 / 编码器 / LED 回流
STM32 数字 IO 回流
```

### 3. GND 与 AGND 的连接方式

推荐连接点：

```text
采样电阻 Rs 下端附近
或 OPA197 / Rs 附近的采样参考点
```
不要在输入模块、Buck 模块、串口屏接口附近把 `GND` 和 `AGND` 相连。

### 4. Rs 采样电阻的 Kelvin 接法

采样电阻既是功率回路的一部分，也是电流检测基准。

主功率路径：

```text
MOS Source → Rs → GND
```

这条路径要粗。

采样路径单独引出：

```text
Rs 上端 Kelvin 点 → OPA197 IN-
Rs 上端 Kelvin 点 → ADC_ISENSE 滤波支路

Rs 下端 Kelvin 点 → AGND_STAR
```

建议理解为：

```text
大电流从 Rs 的功率焊盘走
采样信号从 Rs 焊盘内侧单独拉细线
```


### 5. VOUT+ / VOUT- 采样布局

VOUT+ / VOUT- 采样用于：

```text
负载电压显示
Buck 慢环调节
短路 / 开路判断
```

布局要求：

```text
VOUT+ 采样点从负载接口 VOUT+ 附近引出
VOUT- 采样点从负载接口 VOUT- / MOS Drain 附近引出
两路分压电阻尽量靠近 MCU/模拟采样区域
两路参数、布局尽量对称
分压下端接 AGND
```

不要从功率铜皮中间随便取样。

### 6. Buck 与模拟区的隔离原则

两个 Buck 都是开关电源，是主要噪声源。

Buck 区域包括：

```text
芯片 SW 节点
续流二极管
功率电感
输入电容
输出电容
高 di/dt 回路
```

布局要求：

```text
Buck 放在板子一侧
OPA / DAC / 采样电路放在另一侧
SW 节点面积尽量小
SW 节点下方不要走 ADC / DAC / 运放输入线
功率电感远离 OPA、DAC、采样分压
Buck 输入电容紧贴 Buck VIN 和 GND
续流二极管紧贴 SW 和 GND
```

尤其要避免：

```text
ADC_ISENSE
ADC_VOUT+
ADC_VOUT-
DAC_CH1
DAC_CH2
OPA197 IN+
OPA197 IN-
```

这些线经过 Buck 的 SW 节点、电感、MOS Gate 附近。

### 7. PCB 实施建议

画 PCB 时按区域布局：

```text
左/上侧：输入保护 + Buck 电源区
中间/右侧：MOS + Rs + 负载接口功率区
靠近 Rs：OPA197 + DAC8562 + TS5A3159A + 采样分压
另一侧：STM32、串口屏接口、按键编码器接口
```

推荐原则：

```text
1. 先画功率主回路：VOUT+ → 负载 → MOS → Rs → GND
2. 再画 Rs 的 Kelvin 采样线
3. 再画 OPA/DAC/模拟开关
4. 最后画 MCU、串口、按键、编码器
```

# 调试流程

### 0. 调试前检查

上电前先做以下检查：

```text
1. 检查 35V 输入正负极是否接反。
2. 检查 5V Buck、3.3V LDO 输出是否短路。
3. 检查 AGND 与 GND 是否只通过 0Ω / NetTie 单点连接。
4. 检查 MOS、采样电阻 Rs、负载接口方向是否正确。
5. 检查 DAC_CH2 是否通过模拟开关接到 OPA197 IN+。
6. 检查 OPA197 IN- 是否接到 Rs 上端 Kelvin 采样点。
7. 检查 BUCK_EN 默认下拉，SW_IN 默认下拉。
```

上电默认状态应为：

```text
DAC_CH2 = 0V
SW_IN = 0，模拟开关接 AGND
BUCK_EN = 0，可调 Buck 关闭
输出电流 = 0
```
### 1. 电源部分单独测试

先不要接负载，不要启用恒流环。

#### 1.1 测固定 5V Buck

接入 35V 输入，测量：

```text
5V Buck 输出 ≈ 5V
串口屏接口 5V 正常
OPA197 供电 5V_A 正常
```

注意串口屏供电电流较大，建议确认 5V 不明显跌落。

#### 1.2 测 3.3V LDO

测量：

```text
3.3V 输出 ≈ 3.3V
STM32 核心板供电正常
DAC / 模拟开关供电正常
```

若 3.3V 偏低或 LDO 发热明显，先断开后级排查。


### 2. DAC 与模拟开关测试

先不要接负载，保持可调 Buck 关闭。

#### 2.1 测 DAC_CH2 电流设定通道

用万用表或示波器测 DAC_CH2：

```text
设定 0mA   → DAC_CH2 ≈ 0V
设定 50mA  → DAC_CH2 ≈ 0.1V
设定 100mA → DAC_CH2 ≈ 0.2V
设定 250mA → DAC_CH2 ≈ 0.5V
设定 500mA → DAC_CH2 ≈ 1.0V
```

因为：`V_DAC_CH2=I_set * R_s`
且：`R_s=2Ω`
所以：`500mA \Rightarrow 1.0V`

#### 2.2 测模拟开关

模拟开关接法：

```text
NO  → DAC_CH2
NC  → AGND
COM → R15 → OPA197 IN+
IN  → SW_IN
```

测试：

```text
SW_IN = 0：
COM ≈ 0V，OPA IN+ ≈ 0V

SW_IN = 1：
COM ≈ DAC_CH2，OPA IN+ ≈ DAC_CH2
```

若 SW_IN 未初始化时 COM 接到了 DAC，说明默认状态不安全，需要检查 SW_IN 下拉。

### 3. 恒流环基础测试

这一阶段先**不用可调 Buck 自动调节**。
用实验室电源代替 VOUT+，也就是给功率级提供 V+。

连接方式：

```text
实验室电源正极 → VOUT+
实验室电源负极 → GND
负载接在 VOUT+ 和 VOUT- 之间
```

示波器观察：

```text
CH1：Vsense，也就是 Rs 上端电压
CH2：Vgate，也就是 MOS 栅极电压
```

建议实验室电源先设限流，避免接错时烧器件。

### 4. 恒流精度测试

先接中等负载，例如：

```text
RL = 10Ω 或 25Ω
```

实验室电源 V+ 先设较低值，例如：

```text
V+ = 8V ~ 15V
```

然后按下面顺序测试：

|  设定电流 | 理论 Vsense |
| ----: | --------: |
|   0mA |        0V |
|  50mA |      0.1V |
| 100mA |      0.2V |
| 250mA |      0.5V |
| 500mA |      1.0V |

测试顺序：

```text
1. DAC=0，SW_IN=0，确认 Vsense≈0，Vgate≈0。
2. SW_IN=1，设 50mA，检查 Vsense≈0.1V。
3. 设 100mA，检查 Vsense≈0.2V。
4. 设 250mA，检查 Vsense≈0.5V。
5. 设 500mA，检查 Vsense≈1.0V。
```

如果实际电流明显偏差：

```text
1. 先检查 DAC_CH2 是否输出正确。
2. 再检查 OPA197 IN+ 与 IN- 电压是否接近。
3. 再检查 Rs 是否为 2Ω，焊接是否可靠。
4. 最后做软件标定。
```
### 5. 不同负载测试

分别测试：

```text
10Ω → 25Ω → 50Ω
```

或用开关切换负载。

不同负载下，恒流是否成立的关键是 V+ 是否足够：

V+ >= I * R_L + I * R_s + V_MOS余量

其中：

```text
Rs = 2Ω
VMOS 余量建议 2V~4V
```

因此：
```text
低阻负载：V+ 可以低一些
50Ω 满电流：V+ 需要升到 28V~30V
```

### 6. 稳定性测试

示波器同时观察：

```text
CH1：Vsense
CH2：Vgate
```

重点看是否有持续振荡。

#### 6.1 正常现象

```text
Vsense 稳定在目标值附近
Vgate 无持续高频振荡
负载切换时有短暂过渡，但能恢复稳定
```

#### 6.2 如果 Vsense 有持续高频振荡

优先调整：

```text
1. Rgate 从 47Ω 改到 100Ω
2. 若仍振荡，再焊 C23 = 330pF
3. 若还不够，再尝试 C23 = 1nF
```
#### 6.3 如果只有尖峰毛刺

如果 Vsense 只有负载切换瞬间的尖峰，但没有持续振荡：

```text
1. 先检查探头接地方式，示波器地线不要太长。
2. 检查 Buck 是否离模拟采样区太近。
3. 检查 ADC 采样滤波是否正常。
4. 不建议在 OPA IN- 对地加电容。
```

我们当前方案不放 `IN- 到地电容`，避免让反馈变慢。

#### 6.4 如果响应很慢

如果设定电流变化后，Vsense 很久才到目标值：

```text
1. 检查 C23 是否焊得过大。
2. 检查 Rgate 是否过大。
3. 检查 DAC_CH2 的 RC 滤波是否过慢。
4. 检查软件是否做了过慢的电流爬升。
```
### 7. 可调 Buck 手动测试

在恒流环确认能工作后，再测试可调 Buck。

先不要启用自动 Buck 控制，先让 MCU 输出固定 DAC_CH1，观察 Buck 输出 VOUT+。

你们当前反馈结构近似为：

`V_OUT ≈ 30.98 - 10 * VDAC_CH1`

对应关系：

| DAC_CH1 | VOUT+ 约 |
| ------: | ------: |
|    0.1V |     30V |
|    0.3V |     28V |
|    1.1V |     20V |
|    2.1V |     10V |
|    2.6V |      5V |

注意控制方向：

```text
DAC_CH1 越高 → Buck 输出越低
DAC_CH1 越低 → Buck 输出越高
```

测试时先从低 VOUT 开始：

```text
1. BUCK_EN = 0
2. DAC_CH1 先设为约 2.6V，对应 VOUT+ ≈ 5V
3. 再拉高 BUCK_EN
4. 测 VOUT+ 是否约 5V
5. 再逐步降低 DAC_CH1，让 VOUT+ 升高
```

不要上电默认就让 DAC_CH1=0，否则 Buck 可能直接输出最高电压。
### 8. 自动 Buck 调节测试

自动 Buck 的目标不是调电流，而是控制 MOS 压降。

软件计算： 
`V_MOS=VOUT- - V_sense`

目标：

```text
VMOS ≈ 2V~4V
```

控制逻辑可先用简单滞回：

```text
VMOS < 1.5V：
    V+ 不够，升高 Buck 输出

VMOS > 5V：
    MOS 压降太大，降低 Buck 输出

1.5V ≤ VMOS ≤ 5V：
    保持 Buck 输出
```

调节频率不要太快：

```text
每 50ms ~ 200ms 调整一次
每次只小步改变 DAC_CH1
```

避免 Buck 慢环和运放恒流环互相干扰。

### 9. 保护功能测试

#### 9.1 过流保护

过流阈值：

`I>550mA`

对应：

`V_sense > 0.55A * 2Ω = 1.1V`

执行：

```text
1. SW_IN = 0，模拟开关切到 AGND
2. DAC_CH2 = 0
3. BUCK_EN = 0
4. LCD 显示“过流”
5. alarm_flag 锁存
6. 等待 ALARM_CLEAR 按键清除
```

清除后只回到待机状态，不要自动恢复输出。


#### 9.2 短路保护

短路判断：

```text
Vload = VOUT+ - VOUT- 接近 0
Iactual 接近 Iset
```

短路时不一定要关断输出。题目要求是：

```text
输出端短路时，自动限制输出电流为设定值，且设备不损坏
```

所以建议动作：

```text
1. 保持恒流输出
2. 自动把 Buck 降到较低 V+
3. 显示“短路限流”
4. 短路解除后自动恢复正常 Buck 调节
```

短路测试注意：

```text
先用较低 V+ 测试，例如 5V~8V
不要在 V+=30V 时长时间短路（功耗太大）
```


#### 9.3 开路保护

开路判断：

```text
Iset > 0
Iactual ≈ 0
V+ 已经升到足够高
持续一段时间
```

执行：

```text
1. SW_IN = 0
2. DAC_CH2 = 0
3. BUCK_EN = 0
4. LCD 显示“开路”
5. alarm_flag 锁存
6. 等待 ALARM_CLEAR 按键清除
```

清除后只回到待机状态，不自动恢复输出。

### 10. ADC 采样校准

需要校准的量：

```text
ADC_ISENSE
ADC_VOUT+
ADC_VOUT-
```

#### 10.1 电流采样校准

理论：

·`I_actual=V_sense / 2Ω`

测试方法：

```text
1. 串联万用表测实际电流。
2. 分别设置 50mA、100mA、250mA、500mA。
3. 记录 ADC_ISENSE 和万用表电流。
4. 软件中加入比例校准系数。
```


#### 10.2 VOUT+ / VOUT- 校准

采样电路分压比：
`1 / 11`


测试方法：

```text
1. 用万用表测 VOUT+ 对 GND。
2. 对比 MCU 计算的 VOUT+。
3. 用万用表测 VOUT- 对 GND。
4. 对比 MCU 计算的 VOUT-。
5. 分别得到两个校准系数。
```

### 11. 最终整机测试项目

建议按顺序完成：

```text
1. 5V 电源测试
2. 3.3V 电源测试
3. DAC_CH1 / DAC_CH2 输出测试
4. 模拟开关默认关断测试
5. OPA197 恒流环测试
6. 不同负载恒流测试
7. VOUT+ / VOUT- / Vsense ADC 校准
8. 可调 Buck 手动调压测试
9. 自动 Buck 慢环测试
10. 过流保护测试
11. 短路保护测试
12. 开路保护测试
13. 串口屏显示与报警测试
14. 长时间满载温升测试
```

温升测试重点观察：

```text
MOS
Rs 采样电阻
可调 Buck 电感
5V Buck 电感
LDO
输入保护器件
```

### 12. 安全注意事项

```text
1. 第一次上电时，35V 输入电源必须限流。
2. 第一次测恒流环时，用实验室电源代替可调 Buck 的 V+。
3. 先用低 V+ 测低阻/短路工况。
4. 50Ω、500mA 时再把 V+ 提到 28V~30V。
5. 不要长时间让 MOS 承受十几瓦功耗。
6. BUCK_EN 默认下拉，MCU 初始化完成后再打开。
7. SW_IN 默认下拉，系统默认输出 0A。
8. 过流和开路故障清除后只回到待机，不自动恢复输出。
```

最终调试原则：

```text
先电源，后 DAC；
先低压，后高压；
先手动，后自动；
先恒流环，后 Buck 慢环；
先单项功能，后整机联调。
```
