# GitHub Desktop 开发使用说明

## 推荐使用github desktop，图形化界面易操作

GitHub Desktop 是 GitHub 官方提供的桌面客户端，适合团队协作项目的日常开发。

---

## 先说一个很重要的点

根据 GitHub 官方文档当前说明，**GitHub Desktop 连接 GitHub 时使用的是 HTTPS 认证方式**，而不是直接按命令行里常见的 SSH 工作。

官方说明页：

- [About connections to GitHub in GitHub Desktop](https://docs.github.com/en/desktop/installing-and-authenticating-to-github-desktop/about-connections-to-github-in-github-desktop?apiVersion=2022-11-28)

这意味着：

1. 如果你主要使用 GitHub Desktop，推荐直接登录 GitHub 账号，让 Desktop 帮你完成认证
2. 不建议把“命令行 SSH 工作流”和“GitHub Desktop 工作流”混着理解
3. 如果某个仓库最初是用 SSH 克隆的，GitHub 官方文档提醒你在 GitHub Desktop 中可能会遇到连接问题

所以对这个项目，**如果队友准备用 GitHub Desktop 参与开发，推荐优先使用 GitHub Desktop 自己的登录与克隆流程**。

---

## GitHub Desktop 能帮你做什么

用 GitHub Desktop，队友一般可以完成下面这些事：

1. 登录自己的 GitHub 账号
2. 克隆你们的仓库到本地
3. 查看文件改动
4. 创建分支
5. 提交代码
6. 拉取远程更新
7. 推送自己的修改
8. 发起 Pull Request

对于比赛项目的日常协作，这些功能已经很够用了。

## 安装 GitHub Desktop

官方下载地址：

- [GitHub Desktop Download](https://desktop.github.com/download/)

下载安装后，打开 GitHub Desktop。

## 五、第一次登录 GitHub 账号

第一次使用时，先登录自己的 GitHub 账号。

操作路径：

`File -> Options`

然后在 `Accounts` 页面点击登录按钮。

官方文档：

- [Getting started with GitHub Desktop](https://docs.github.com/en/desktop/overview/getting-started-with-github-desktop?platform=windows)

官方截图：

![GitHub Desktop 登录界面](https://docs.github.com/assets/cb-65983/images/help/desktop/windows-sign-in-github.png)

登录成功后，GitHub Desktop 会和你的 GitHub 账号关联，后续克隆、拉取、推送会更顺畅。

## 把仓库克隆到本地

对于项目，推荐两种方式。

### 方式 1：在 GitHub 网页中打开仓库，再用 GitHub Desktop 克隆

进入仓库主页后，点击绿色的 `Code` 按钮。

官方截图：

![GitHub 仓库页面中的 Code 按钮](https://docs.github.com/assets/cb-13128/images/help/repository/code-button.png)

然后点击 `Open with GitHub Desktop`。

官方截图：

![从 GitHub 网页用 GitHub Desktop 打开仓库](https://docs.github.com/assets/cb-44807/images/help/repository/open-with-desktop.png)

之后 GitHub Desktop 会弹出克隆窗口，让你选择本地保存位置。

官方文档：

- [Cloning a repository from GitHub to GitHub Desktop](https://docs.github.com/en/desktop/adding-and-cloning-repositories/cloning-a-repository-from-github-to-github-desktop)

### 方式 2：直接在 GitHub Desktop 里克隆

在 GitHub Desktop 中点击：

`File -> Clone repository`

官方截图：

![GitHub Desktop 中的 Clone repository 菜单](https://docs.github.com/assets/cb-10969/images/help/desktop/clone-file-menu-windows.png)

接着在弹出的窗口中选择仓库并设置本地路径。

官方截图：

![GitHub Desktop 的克隆仓库窗口](https://docs.github.com/assets/cb-29400/images/help/desktop/choose-repository-location-mac.png)

官方文档：

- [Cloning and forking repositories from GitHub Desktop](https://docs.github.com/en/desktop/adding-and-cloning-repositories/cloning-and-forking-repositories-from-github-desktop)

## 开始开发前先同步最新代码

每次开始工作前，建议先检查远程有没有新内容。

在 GitHub Desktop 顶部会看到 `Fetch origin` 按钮。

官方截图：

![Fetch origin 按钮](https://docs.github.com/assets/cb-58337/images/help/desktop/fetch-button.png)

点击后，如果远程有更新，通常会变成 `Pull origin`，这时再把更新拉下来。

官方文档：

- [Syncing your branch in GitHub Desktop](https://docs.github.com/en/desktop/working-with-your-remote-repository-on-github-or-github-enterprise/syncing-your-branch-in-github-desktop)

建议养成习惯：

1. 打开项目先点一次 `Fetch origin`
2. 如果有更新，再执行 `Pull origin`
3. 同步完成后再开始写代码

## 修改文件时，GitHub Desktop 会自动显示改动

真正编辑文件一般不在 GitHub Desktop 里做，而是在：

- VS Code
- Keil
- 记事本
- 其他你熟悉的编辑器

你只需要保存文件，GitHub Desktop 就会自动检测到变更。

左侧 `Changes` 面板会列出被修改的文件。

官方截图：

![Changes 面板会显示改动文件](https://docs.github.com/assets/cb-16555/images/help/desktop/commit-all.png)

你可以：

1. 查看哪些文件被改了
2. 勾选这次想提交的文件
3. 不勾选暂时不想提交的内容

官方文档：

- [Committing and reviewing changes to your project in GitHub Desktop](https://docs.github.com/en/desktop/making-changes-in-a-branch/committing-and-reviewing-changes-to-your-project-in-github-desktop?platform=windows)

## 写提交信息并提交

当你确认改动没问题后，在 GitHub Desktop 左下角填写提交信息：

- `Summary`：简短说明这次改了什么
- `Description`：可选，补充更多细节

推荐写法：

- `新增串口调试指令解析`
- `修复 ADC 采样缩放问题`
- `补充项目协作说明文档`

不推荐写法：

- `update`
- `改了`
- `123`

填写后点击 `Commit to 当前分支名`。

这一步只是提交到**本地仓库**，还没有上传到 GitHub。

---

## 把本地提交推送到 GitHub

本地提交后，顶部一般会出现 `Push origin` 或 `Publish branch`。

### 如果这是已有远程分支

点击 `Push origin` 即可把提交推到 GitHub。

官方截图：

![Push origin 按钮](https://docs.github.com/assets/cb-17787/images/help/desktop/push-to-origin.png)

### 如果这是你新建的本地分支，远程还没有

先点击 `Publish branch`，把这个分支发布到 GitHub。

官方截图：

![Publish branch 按钮](https://docs.github.com/assets/cb-29970/images/help/desktop/publish-branch-button.png)

只有分支先发布到 GitHub，队友和网页端才能看到它。

## 日常开发的完整流程

下面这套流程最适合你们现在这个项目。

### 每次开始开发

1. 打开 GitHub Desktop
2. 选中项目仓库
3. 确认当前仓库正确
4. 点击 `Fetch origin`
5. 如果有更新，点击 `Pull origin`
6. 确认自己在正确分支上
7. 如果要做新功能，创建新分支

### 开发中

1. 用编辑器修改代码
2. 回 GitHub Desktop 查看变更
3. 检查这次改动是否合理
4. 填写提交信息
5. 点击 `Commit to ...`

### 开发后

1. 点击 `Push origin` 或 `Publish branch`
2. 如果是功能分支，发起 Pull Request
3. 通知队友你已经提交了什么内容

## 使用 GitHub Desktop 时的注意事项

### 1. 开发前先同步

不要一打开仓库就直接改，先看远程有没有更新。

### 2. 尽量按模块分支开发

不要所有人一直直接在 `main` 上写。

### 3. 提交信息写清楚

这样以后回顾历史非常省事。

### 4. 改核心文件前先在群里说一声

比如：

- 主流程
- 协议定义
- 硬件接口
- 公共头文件

这些内容多人同时改时最容易冲突。

### 5. 不懂冲突时先别乱点

如果 GitHub Desktop 提示冲突，先找队友确认一下要保留哪部分内容，不要盲目覆盖。

## 最简工作流

如果只记最少步骤，可以记下面这套：

1. 打开 GitHub Desktop
2. 选中仓库
3. 点 `Fetch origin`
4. 有更新就点 `Pull origin`
5. 新建分支
6. 改文件
7. 回 GitHub Desktop 查看改动
8. 写提交说明并 `Commit`
9. 点击 `Push origin` 或 `Publish branch`
10. 需要合并时发 Pull Request
